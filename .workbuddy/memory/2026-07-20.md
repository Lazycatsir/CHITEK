## 【关键修复】reveal 动画"标题可见但不动"根因 = Astro scoped :global 坑
- 症状：标题一直 opacity:1、无动画。用 Playwright 无头浏览器实测拿到铁证：`.revealed` 类的 JS toggle 完全正常（可重复），但未 revealed 时 computed opacity 仍是 1 → 基础隐藏态 CSS 没生效。
- 根因：Astro scoped `<style>` 会给选择器**每一段**加 `[data-astro-cid-xxx]`，包括写在 html 上的全局类 `.js`，编译成 `.js[data-astro-cid-xxx] .series-heading[data-astro-cid-xxx]{opacity:0}`。但 `<html>`(带 .js)在组件外、没有 cid 属性 → `.js[cid]` 永不匹配 → 整条隐藏态失效。
- 修复：把跨组件的全局类用 `:global()` 包起来排除 scope：`:global(.js) .series-heading{...}`。Astro 只对 `.series-*` 加 cid，`.js` 保持全局，编译成 `.js .series-heading[cid]` 正确匹配。
- 验证方式（可复用）：Playwright(共享 venv python) headless 加载 → scrollIntoView → 读 getComputedStyle(opacity/transform) + classList，来回滚动验证可重复。脚本在 %TEMP%\ahf_anim_test.py。
- 教训：Astro `<style>` 里凡是引用组件外元素的类（html/body 上的全局 flag 类，如 .js/.dark/.rtl），必须用 `:global()`，否则被 scope 后永不匹配。

## 【再次修复】标题"看不见动画"的真实根因 = 整组同时触发，标题先经过已播完
- 背景：修好 :global 坑 + .js 守卫后，Playwright 实测标题 opacity 0→1、translateX -40→0、可重复触发**全部正常**。但用户仍反馈"只有卡片正常、标题不动"。
- 真实根因（step-scroll Playwright 实测定位）：之前的动画是**整个 `.reveal-group` 一进入视口就同时给所有子元素加 `.revealed`**。标题在 group 最顶部、用户向下滚时最先经过视口，动画（delay 0s）在用户还没注意时就播完了；而卡片在下方、delay 分布宽（0.2–0.68s），用户滚到时正赶上动画 → 视觉上"标题没动、卡片有"。
- 修复：把"整组触发"改为**每个元素独立 IntersectionObserver**——`.series-heading`/`.series-subtitle`/`.series-card` 各自进入视口时才加 `.revealed`（离开移除，可重复）。CSS 显示态从 `.reveal-group.revealed .series-xxx` 改为 `.series-xxx.revealed`；卡片 stagger delay 由 JS 按组内 index 设 inline transitionDelay。
- 验证（step-scroll 实测）：标题初始 opacity=0/translateX=-40px/rectTop=3034(视口外)；向下滚到 scrollY=2250 时标题 rectTop=784（仍在视口内 900 高）才**首次触发** revealed，opacity 过渡 0.99→1、translateX→0；滚回顶部重置 opacity=0。→ 标题在用户看得见它时滑入，效果可见且可重复。
- 教训：长区块内的"标题+多卡"若共用一个 group 触发，标题往往早于用户注意就动画完；逐元素触发才能让每个元素在其可见时才动。
